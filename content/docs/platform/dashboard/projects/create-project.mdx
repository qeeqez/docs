Every workspace in **RIXL** begins with a project.  
A project organizes your media, permissions, and API access in one place.

<Callout type="info">
Projects are workspaces with dedicated storage, team access, and API keys.  
You must create at least one project before uploading media.
</Callout>

---

## Step-by-Step Setup

<Steps>
  <Step>
    **Access Project Creation**  
    From your dashboard, go to **Settings → Projects** and click **Create New Project**.
  </Step>

  <Step>
    **Configure Project Settings**  

    - **Name** → Choose a descriptive name (e.g., “Brand X Campaign”).  
    - **Region** → Pick the server closest to your audience for best performance.  
    - **Video Quality** → Select a processing tier (see below).  

    <Callout type="warn">
      Server region cannot be changed after project creation.
    </Callout>
  </Step>

  <Step>
    **Confirm Setup**  
    Review your choices and click **Create Project**. Most settings can be adjusted later in [Project Settings](./project-settings).
  </Step>
</Steps>

## Video Processing Tiers

| Tier       | Quality          | Resolution  | Processing Speed | Best For              |
|------------|------------------|-------------|------------------|-----------------------|
| **Basic**  | Standard         | Up to 720p  | Standard         | Development & Testing |
| **Shorts** | Mobile Optimized | Up to 1080p | Fast             | Social Media Content  |
| **Pro**    | Professional     | Up to 4K    | Priority         | Video-on-Demand       |

<Callout type="info">
See full details in the [Video Processing Guide](/platform/uploads/videos/quality-tiers).
</Callout>
