## What are Projects on RIXL Cloud?

Projects help you organize your media, manage team access, and track usage.
You'll need to create a project before uploading any media.

<Callout type="info">
    Projects serve as workspaces that contain all your content with dedicated settings and API keys.
    Once a project is set up, you can create API keys for it and move forward with media uploads and management.
</Callout>

## Creating Your Project

[//]: # ([UI WALKTHROUGH: Step-by-step visual guide for project creation process])

<Steps>
    <Step>
        ### Access Project Creation

        From your dashboard, navigate to **Settings â†’ Projects** and click **Create New Project**.
        
        [//]: # ([IMAGE NEEDED: Screenshot of project creation interface])
    </Step>

    <Step>
        ### Configure Project Settings

        **Project Name**: Choose a descriptive name for easy identification

        **Server Location**: Select your preferred region (default: **EU Central - Germany**)
        
        **Video Quality**: Choose optimization level:
        - **Basic**: Testing and development
        - **Shorts**: Social content and vertical videos (recommended)
        - **Pro**: Professional video-on-demand content
        
        <Callout type="info">
            For detailed tier specifications, see [Video Processing documentation](/docs/platform/uploads/video-processing/quality-tiers).
        </Callout>

        <Callout type="warn">
            Server location affects performance for your users. Choose the region closest to your audience.
        </Callout>
    </Step>

    <Step>
        ### Complete Setup

        Review your settings and click **Create Project**. You can modify most settings later, but server location cannot be changed.
    </Step>
</Steps>

## Project Configuration Options

<Tabs items={['Storage & Processing', 'Video Optimization', 'Security & Access']}>
    <Tab value="Storage & Processing">
        ### Server Locations
        
        Choose your preferred data center region based on your audience location:
        
        - **EU Central - Germany**: European users and GDPR compliance
        - **US East**: North American users  
        - **Asia Pacific**: Asian markets
        
        <Callout type="tip">
            Server location affects performance for your users. Choose the region closest to your audience.
        </Callout>
        
        ### Processing Settings
        
        - **Automatic transcoding**: All video formats optimized for web delivery
        - **Image optimization**: Smart compression and format selection
        - **CDN distribution**: Global content delivery network
        - **Real-time processing**: Live status updates during upload
    </Tab>
    
    <Tab value="Video Optimization">
        ### Video Processing Overview
        
        Choose the right processing tier for your content:
        
        - **Basic**: Standard quality, up to 720p resolution
        - **Shorts**: Mobile optimized, up to 1080p resolution  
        - **Pro**: Professional quality, up to 4K resolution
        
        ### Key Features by Tier
        
        - **Basic**: Thumbnail generation, standard processing
        - **Shorts**: Fast processing, mobile-first optimization, adaptive streaming
        - **Pro**: Multiple quality levels, advanced analytics, priority processing
        
        <Callout type="info">
            For complete specifications and feature comparisons, see [Video Processing documentation](/docs/platform/uploads/video-processing).
        </Callout>
    </Tab>
    
    <Tab value="Security & Access">
        ### Project Visibility
        
        - **Organization**: Available to all organization members
        - **Team-specific**: Restricted to assigned team members
        - **Private**: Owner access only
        
        ### Access Control Features
        
        - **API key scoping**: Project-specific authentication
        - **Team member management**: Role-based access control
        - **Invitation controls**: Managed team invitations
        - **Usage monitoring**: Track access and API usage
        
        ### Security Settings
        
        - **CORS configuration**: Cross-origin resource sharing setup
        - **Rate limiting**: API request throttling
        - **Webhook security**: Signed webhook payloads
        - **Access logs**: Detailed activity tracking
    </Tab>
</Tabs>
