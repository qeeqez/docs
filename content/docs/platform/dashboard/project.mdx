<Callout type="info">
You need at least one project to use the main dashboard. If you don’t have one, you’ll be prompted to create it on first login.
</Callout>

## Create a New Project

<Steps>
  <Step>
  **Project Details**  
  Navigate to **Settings → Projects → Create New Project** and provide:
  - **Name**: e.g., “Marketing Videos” or “Product Photos”
  - **Description**: Optional context for your team
  </Step>

  <Step>
  **Server Location**  
  Choose the region closest to your users for best performance:
  - **US East** - North America, Eastern regions  
  - **US West** - Pacific regions, Asia  
  - **Europe** - Europe, Africa  

  <Callout type="warning">
  Server location cannot be changed after creation.
  </Callout>
  </Step>

  
  <Step>
  **Optimization Settings**
  Select a default video quality tier based on your content type:
  - **Shorts** – Social media & mobile  
  - **Standard** – Balanced web delivery  
  - **Pro** – Premium VOD & cinematic  
  </Step>
  
  <Step>
    **Create & Start Uploading**
    
    Once created, you can immediately start [uploading media](/platform/uploads/overview) and [generating API keys](/platform/dashboard/api-keys).
  </Step>
</Steps>

## Project Configuration

### Optimization Settings

Configure how RIXL automatically optimizes media, and you can adjust default:

**Video**  
- Quality tiers (Shorts, Standard, Pro)  
- Automatic thumbnails  
- MP4 + adaptive HLS support  

**Image**  
- Automatic AVIF/WEBP conversion  
- Thumbhash placeholders  
- On-the-fly transformations  

### Access Control
Roles define what team members can do:
- **Owner** - Full control (billing + deletion)  
- **Admin** - Manage settings, content, and members  
- **Editor** - Upload & edit content  
- **Viewer** - Read-only access  

**API Access**  
Generate project-specific keys with granular permissions. See [API Access docs](/platform/account/api-access/overview).

### Usage Monitoring
Track real-time consumption:  
- Storage, bandwidth, and API call volume  
- Processing queue and job stats  
- Team activity and contribution logs

### API Key Creation Process

<Steps>
<Step>
**Access Quick Actions**

Click **Create API Key** in the Quick Actions panel
</Step>
<Step>
**Configure Settings**

<Tabs items={['Key Information', 'Security Settings']}>
<Tab value="Key Information">
- **API Key Name**: Descriptive identifier
- **Project Association**: Pre-filled with current project
- **Purpose Documentation**: Internal notes for usage tracking
</Tab>
<Tab value="Security Settings">
**Expiration Options:**
- **Forever**: Permanent access
- **1 Week**: Short-term testing
- **1 Year**: Long-term production
- **Custom Duration**: Specific date-based expiration
</Tab>
</Tabs>
</Step>
<Step>
**Generate and Secure Key**

Click **Create** to generate, then immediately copy and store securely
</Step>
</Steps>

<Callout type="error">
    **Security Critical**: API keys are only displayed once during creation. Copy and store immediately as they cannot be retrieved later.
</Callout>

## Best Practices

<Callout type="tip">
Well-organized projects enables team collaborate more efficiently and makes scaling easier.
</Callout>

**Organization Principles:**
- Use descriptive, consistent project names
- Separate environments into distinct projects
- Review and update team permissions quarterly
- Archive or delete unused content regularly
- Document project purpose clearly
