Content [feeds](https://api.rixl.com/feeds) provide structured organization and distribution of your media content within projects. 
This guide covers creating and managing feeds for dynamic content delivery.

<Callout type="info">
Feeds are project-specific content collections. Make sure you have 
[created a project](/docs/getting-started/create-project) before working with feeds.
</Callout>

## Creating Feeds

### Dashboard Creation Method

<Steps>
<Step>
**Access Feed Creation**

Navigate to the [Feeds section](https://api.rixl.com/feeds) from the sidebar and click **Create Feed**
</Step>
<Step>
**Select Feed Type**

Choose from available feed types based on your content strategy and use case
</Step>
<Step>
**Configure Feed Settings**

<Tabs items={['Basic Settings', 'Visibility Options']}>
<Tab value="Basic Settings">
- **Feed Name**: Enter a descriptive, memorable name
- **Feed Description**: Add optional detailed description
- **Project**: Select the project this feed belongs to
</Tab>
<Tab value="Visibility Options">
- **Public**: Accessible to anyone with the feed URL
- **Unlisted**: Accessible with direct link but not discoverable
- **Private**: Restricted access with permission controls
</Tab>
</Tabs>
</Step>
<Step>
**Finalize Creation**

Review your settings and click **Create** to generate your new feed
</Step>
</Steps>

### API Creation Method

For programmatic feed creation, use the dashboard or contact support for API access to feed management endpoints.

<Callout type="info">
Feed creation is currently available through the dashboard interface. Programmatic feed management will be available in future API versions.
</Callout>

### Content Addition Methods

<Tabs items={['Manual Addition', 'Bulk Operations', 'Dynamic Rules']}>
<Tab value="Manual Addition">
<Steps>
<Step>Navigate to the feed dashboard and click **Add Content**</Step>
<Step>Browse or search through your media library</Step>
<Step>Select individual items to add to the feed</Step>
<Step>Arrange content order using drag-and-drop interface</Step>
<Step>Click **Add to Feed** to confirm additions</Step>
</Steps>
</Tab>
<Tab value="Bulk Operations">
<Steps>
<Step>Navigate to the feed dashboard and click **Bulk Add**</Step>
<Step>
**Use Advanced Filters:**
- Content type filtering
- Date range selection
- Tag-based filtering
- Performance metrics filtering
</Step>
<Step>Select multiple items using checkboxes</Step>
<Step>Click **Add Selected** to add all items simultaneously</Step>
</Steps>
</Tab>
<Tab value="Dynamic Rules">
**Automated Dynamic Rules:**
- **Content Type**: Specify video, image, or mixed media
- **Tags & Categories**: Include content with specific metadata
- **Date Criteria**: Set upload date ranges or recency rules
- **Performance Thresholds**: Include based on view counts or engagement
- **Ordering Rules**: Automatic sorting by newest, most viewed, trending
</Tab>
</Tabs>

### Content Reordering

<Steps>
<Step>Navigate to your feed dashboard</Step>
<Step>Click **Reorder** to enter reorder mode</Step>
<Step>Drag items to desired positions using the drag handles</Step>
<Step>Click **Save Order** to apply changes</Step>
</Steps>

<Callout type="tip">
Schedule content to appear or disappear from feeds automatically to maintain fresh, timely content delivery.
</Callout>

## Feed Integration & Distribution

### Website Embedding

<Steps>
<Step>
**Access Embed Options**

Navigate to your feed dashboard and click the **Embed** button
</Step>
<Step>
**Configure Display Settings**

Customize embedded feed appearance:
- Width and height dimensions
- Theme and color scheme
- Interactive features (likes, comments, sharing)
- Loading behavior and pagination
</Step>
<Step>
**Generate Embed Code**

Copy the provided iframe code for your website integration
</Step>
<Step>
**Implement on Website**

Paste the embed code into your website's HTML where you want the feed to appear
</Step>
</Steps>

**Example Embed Code:**
```html
<iframe 
  src="https://feeds.rixl.com/embed/feed_123456" 
  width="100%" 
  height="600" 
  frameborder="0"
  allowfullscreen>
</iframe>
```

### API Integration

```bash
curl -X GET https://api.rixl.com/v1/feeds/{feedId} \
     -H "X-API-Key: YOUR_PROJECT_API_KEY"
```

## Troubleshooting Guide

<Tabs items={['Display Issues', 'Content Problems', 'Integration Issues']}>
<Tab value="Display Issues">
<Steps>
<Step>
**Check Visibility Settings**

If feed not displaying, verify visibility settings and access permissions
</Step>
<Step>
**Optimize Performance**

For slow loading, reduce items per page or optimize content sizes
</Step>
<Step>
**Verify Embedding Permissions**

For embedding issues, check domain whitelist and embedding permissions
</Step>
</Steps>
</Tab>
<Tab value="Content Problems">
<Steps>
<Step>
**Verify Content Criteria**

If content missing from feed, check if content meets feed criteria or was properly added
</Step>
<Step>
**Check Ordering Settings**

For wrong content order, verify manual vs. automatic ordering settings
</Step>
<Step>
**Review Dynamic Rules**

For outdated content, check dynamic rules and content freshness settings
</Step>
</Steps>
</Tab>
<Tab value="Integration Issues">
<Steps>
<Step>
**Verify API Permissions**

For API access problems, check API key permissions and rate limits
</Step>
<Step>
**Check RSS Settings**

For RSS issues, verify RSS output settings and cache refresh
</Step>
<Step>
**Validate HTML**

For embed code issues, validate HTML and check browser console for errors
</Step>
</Steps>
</Tab>
</Tabs>

## Best Practices

<Callout type="tip">
Well-organized feeds significantly improve user engagement and content discoverability.
</Callout>

**Organization Principles:**
- Create feeds with related, coherent content themes
- Update and curate feed content consistently
- Balance content richness with loading speed
- Maintain consistent thumbnail quality and style
- Add comprehensive metadata for better discoverability

**Performance Optimization:**
- Limit initial feed size for faster loading
- Use lazy loading for large feeds
- Optimize images and video thumbnails
- Implement caching strategies
- Monitor and optimize based on analytics
