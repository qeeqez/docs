## Create a New Project

<Steps>
  <Step>
    **Project Details**
    
    Navigate to **Settings → Projects → Create New Project** and provide:
    - **Project Name**: Something descriptive (e.g., "Marketing Videos", "Product Photos")
    - **Description**: Optional context for team members
  </Step>
  
  <Step>
    **Server Location**
    
    Choose the region closest to your users for optimal performance:
    - **US East**: Best for North America and Eastern regions
    - **US West**: Ideal for Pacific regions and Asia
    - **Europe**: Optimal for European and African users
    
    <Callout type="warning">
      Server location cannot be changed after project creation—choose carefully!
    </Callout>
  </Step>
  
  <Step>
    **Optimization Settings**
    
    Select default video quality based on your content type:
    - **Shorts**: Optimized for social media and mobile viewing
    - **Standard**: Balanced quality for general web content  
    - **Pro**: High quality for professional and cinematic content
  </Step>
  
  <Step>
    **Create & Start Uploading**
    
    Once created, you can immediately start [uploading media](/docs/uploads) and [generating API keys](/docs/dashboard/api-keys).
  </Step>
</Steps>

## Project Configuration

### Optimization Settings

Configure how RIXL processes your media:

**Video Settings**
- **Quality Tiers**: Shorts, Standard, or Pro transcoding
- **Thumbnail Generation**: Automatic poster frames and previews
- **Format Support**: MP4, HLS adaptive streaming

**Image Settings**
- **Format Conversion**: Automatic AVIF optimization
- **Placeholder Generation**: Instant-loading thumbhash previews
- **Transformation**: Real-time URL-based image editing

### Access Control

**Project Permissions**
- **Owner**: Full control including deletion and billing
- **Admin**: Manage content, settings, and team members
- **Editor**: Upload and edit content
- **Viewer**: Read-only access to content

**API Access**
- Generate project-specific API keys
- Set granular permissions per key
- Monitor usage and rate limits

### Usage Monitoring

Track your project's resource consumption:

- **Storage Usage**: Total media storage across all formats
- **Bandwidth**: Monthly delivery and processing usage
- **API Calls**: Request volume and rate limit status
- **Team Activity**: Member actions and upload history

### API Key Creation Process

<Steps>
<Step>
**Access Quick Actions**

Click **Create API Key** in the Quick Actions panel
</Step>
<Step>
**Configure Settings**

<Tabs items={['Key Information', 'Security Settings']}>
<Tab value="Key Information">
- **API Key Name**: Descriptive identifier
- **Project Association**: Pre-filled with current project
- **Purpose Documentation**: Internal notes for usage tracking
</Tab>
<Tab value="Security Settings">
**Expiration Options:**
- **Forever**: Permanent access
- **1 Week**: Short-term testing
- **1 Year**: Long-term production
- **Custom Duration**: Specific date-based expiration
</Tab>
</Tabs>
</Step>
<Step>
**Generate and Secure Key**

Click **Create** to generate, then immediately copy and store securely
</Step>
</Steps>

<Callout type="error">
    **Security Critical**: API keys are only displayed once during creation. Copy and store immediately as they cannot be retrieved later.
</Callout>

## Best Practices

<Callout type="tip">
Well-organized projects enable teams to collaborate more efficiently and improve content discoverability.
</Callout>

**Organization Principles:**
- Use descriptive, consistent project names
- Group related media content within projects
- Review and update team permissions quarterly
- Archive or delete unused content regularly
- Maintain clear project descriptions and guidelines
