### API Key Creation Process

<Steps>
<Step>
**Access Quick Actions**

Click **Create API Key** in the Quick Actions panel
</Step>
<Step>
**Configure Settings**

<Tabs items={['Key Information', 'Security Settings']}>
<Tab value="Key Information">
- **API Key Name**: Descriptive identifier
- **Project Association**: Pre-filled with current project
- **Purpose Documentation**: Internal notes for usage tracking
</Tab>
<Tab value="Security Settings">
**Expiration Options:**
- **Forever**: Permanent access
- **1 Week**: Short-term testing
- **1 Year**: Long-term production
- **Custom Duration**: Specific date-based expiration
</Tab>
</Tabs>
</Step>
<Step>
**Generate and Secure Key**

Click **Create** to generate, then immediately copy and store securely
</Step>
</Steps>

<Callout type="error">
    **Security Critical**: API keys are only displayed once during creation. Copy and store immediately as they cannot be retrieved later.
</Callout>


## Team Management System

<Steps>
<Step>
**Access Team Management**

Navigate to **Project Settings → Team**
</Step>
<Step>
**Invite Team Members**

<Tabs items={['Invitation Process', 'Role Selection', 'Requirements']}>
<Tab value="Invitation Process">
1. Click **Invite Team Member**
2. Enter registered email address
3. Select appropriate role and permissions
4. Send an invitation with a personalized message
5. Track invitation status
</Tab>
<Tab value="Role Selection">
- **Admin**: Full project management access
- **Editor**: Content creation, modification, and feed management
- **Viewer**: Read-only access to project resources
</Tab>
<Tab value="Requirements">
- **Account Requirement**: Invitees must have existing RIXL accounts
- **Ownership Structure**: Project creators automatically receive Owner role
- **Permission Hierarchy**: Owners can assign and modify all roles
</Tab>
</Tabs>
</Step>
<Step>
**Manage Team Collaboration**

Monitor team activity, adjust permissions, and facilitate collaboration
</Step>
</Steps>

## Best Practices

<Callout type="tip">
Well-organized projects enable teams to collaborate more efficiently and improve content discoverability.
</Callout>

**Organization Principles:**
- Use descriptive, consistent project names
- Group related media content within projects
- Review and update team permissions quarterly
- Archive or delete unused content regularly
- Maintain clear project descriptions and guidelines
