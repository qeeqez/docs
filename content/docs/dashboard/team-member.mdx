## Roles and Permissions

### Organization-Level Roles

<Tabs items={['Owner', 'Admin', 'Member']}>
<Tab value="Owner">
**Owner Role - Maximum Authority**
- Complete control over organization settings
- Full billing and subscription management
- Add, remove, and manage all members
- Create, modify, and delete any project
- Access to all organization features

<Callout type="info">
The account creator is automatically assigned as Owner.
</Callout>
</Tab>
<Tab value="Admin">
**Admin Role - Administrative Functions**
- Create, configure, and manage projects
- Invite and manage team members (excluding Owner)
- Configure organization settings
- Oversee content across projects
- Cannot modify billing or remove owners
</Tab>
<Tab value="Member">
**Member Role - Project Participation**
- Access to specifically assigned projects
- Create and manage content within permitted projects
- Full collaboration capabilities within boundaries
- Cannot create projects or manage organization settings
</Tab>
</Tabs>

### Project-Level Roles

<Cards>
  <Card title="Project Owner" description="Complete project authority">
    - Complete authority over all project aspects
    - Add, remove, and manage project team members
    - Configure all project settings
    - Authority to delete the entire project
  </Card>
  
  <Card title="Editor" description="Content creation and management">
    - Upload, modify, and organize project content
    - Create and manage content feeds
    - Access to project analytics
    - Full collaboration features
  </Card>
  
  <Card title="Viewer" description="Read-only access">
    - View all project content and feeds
    - Read-only access to project metrics
    - Browse project resources
    - Cannot modify, upload, or delete content
  </Card>
</Cards>

## Team Member Invitations

### Organization-Level Invitations

<Steps>
  <Step>
    Navigate to **[Organization Settings → Team](https://api.rixl.com/settings/team/general)** → **[Invite Team Member](https://api.rixl.com/settings/team/members)**
  </Step>
  <Step>
    Configure invitation details:
    - Email address and personal message
    - Role assignment (Admin/Member)
    - Optional project access selection
  </Step>
  <Step>
    Send and track invitation through dashboard
  </Step>
</Steps>

### Project-Level Invitations

<Steps>
  <Step>
    Open project → **Project Settings → Team** or **Quick Actions → Invite Team Member**
  </Step>
  <Step>
    Configure project invitation:
    - Email address and project context
    - Role selection (Editor/Viewer)
    - Duration settings if supported
  </Step>
  <Step>
    Send project-specific invitation
  </Step>
</Steps>

### Invitation Requirements

<Callout type="warning">
Team members must already have registered RIXL Cloud accounts before invitation.
</Callout>

**Prerequisites:**
- Pre-existing RIXL account required
- Verified email address
- Active account status
- Project creator automatically becomes Owner

**Workflow:**
- System validates existing account before sending
- Email notification sent to valid accounts
- Invitations remain pending until accepted/declined
- Invitations expire after 7 days

<Callout type="info">
Users without existing RIXL accounts must sign up first before receiving invitations.
</Callout>

### Role Modification

**Organization Roles:**
<Steps>
  <Step>
    Go to **Organization Settings → Team → Members**
  </Step>
  <Step>
    Select member requiring role change
  </Step>
  <Step>
    Click **Change Role** and select new role
  </Step>
  <Step>
    Confirm change and notify member
  </Step>
</Steps>

**Project Permissions:**
<Steps>
  <Step>
    Access **Project Settings → Team**
  </Step>
  <Step>
    Locate member requiring changes
  </Step>
  <Step>
    Click **Edit Permissions** for detailed controls
  </Step>
  <Step>
    Modify role and save changes
  </Step>
</Steps>

### Member Removal

<Callout type="error">
Member removal is permanent and immediately revokes all access permissions.
</Callout>

<Tabs items={['Organization Removal', 'Project Removal', 'Bulk Operations']}>
<Tab value="Organization Removal">
<Steps>
  <Step>
    Navigate to **Organization Settings → Team → Members**
  </Step>
  <Step>
    Select member to remove
  </Step>
  <Step>
    Click **Remove from Organization** and confirm
  </Step>
  <Step>
    Member loses access to all organization resources
  </Step>
</Steps>
</Tab>
<Tab value="Project Removal">
<Steps>
  <Step>
    Access **Project Settings → Team**
  </Step>
  <Step>
    Find member to remove from project
  </Step>
  <Step>
    Click **Remove from Project** and confirm
  </Step>
  <Step>
    Member retains organization access but loses project permissions
  </Step>
</Steps>
</Tab>
<Tab value="Bulk Operations">
- Bulk selection for simultaneous operations
- Apply role modifications to multiple members
- Modify project access for member groups
- Send group notifications about changes
</Tab>
</Tabs>

## Troubleshooting

<Tabs items={['Invitation Problems', 'Access Issues', 'Permission Conflicts']}>

## Best Practices

<Callout type="tip">
Teams with defined roles boost collaboration and reduce administrative inefficiency.
</Callout>

**Organization Strategy:**
- Clear role definitions and responsibilities
- Minimum necessary permission assignment
- Project-based organization for a limited scope
- Regular quarterly role assessments

**Security Management:**
- Regular permission audits
- Prompt inactive member removal
- Role appropriateness evaluation
- Access pattern monitoring
