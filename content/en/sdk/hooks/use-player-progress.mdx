---
title: "usePlayerProgress"
description: "Read timeline state and perform seeking without reaching for a DOM video ref."
---

`usePlayerProgress(playerId)` is the preferred public hook for custom timeline UI with the public `Video` component.

## Signature

```tsx
const progress = usePlayerProgress(playerId);
```

It returns:

- `currentTime`
- `duration`
- `progress`
- `buffered`
- `seekTo(seconds)`
- `seekBy(deltaSeconds)`

`progress` and `buffered` are ratios from `0` to `1`.

## Example

```tsx
import {Video, usePlayerProgress} from "@rixl/videosdk-react";

const playerId = "00000000-0000-0000-0000-0000000000e5";

function CustomProgressControls() {
  const {currentTime, duration, progress, buffered, seekBy} = usePlayerProgress(playerId);

  return (
    <div className="space-y-3">
      <Video
        id="your-video-id"
        playerId={playerId}
        theme="hideUI"
        analytics={false}
      />

      <div className="space-y-2">
        <div>{`${Math.round(currentTime)}s / ${Math.round(duration)}s`}</div>
        <div>{`Played: ${Math.round(progress * 100)}% | Buffered: ${Math.round(buffered * 100)}%`}</div>
        <button type="button" onClick={() => seekBy(10)}>
          Seek +10s
        </button>
      </div>
    </div>
  );
}
```

## Notes

- This hook works with the public `Video` component by matching on `playerId`.
- Before metadata is ready, timeline values can be `0`.
- Prefer this hook over `useProgressBar` when you are building on top of the public `Video` component.
